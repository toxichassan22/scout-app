# Scoring System — App Scout

## نظرة عامة

نظام التحكيم والتقييم لمسابقات المهرجان. الحكام بيقيّموا تقارير الفرق، والدرجات بتتجمع في leaderboard عند الأدمن.

---

## ⚠️ قواعد مهمة

- **الأنشطة خارج التقييم** — الأنشطة (color hunt, easter egg, hack the bank, code guess, QR puzzle) للترفيه خلال اليوم **ومالهاش بنود تقييم ولا درجات**.
- **المسابقات بس** هي اللي ليها بنود تقييم (عبقرينو، حقيقتين وكذبة، تصميم فيديو، العاصمة والعملة... إلخ).
- **الأدمن هو اللي بيتحكم في موعد إقلاع الأنشطة** (manual toggle زي المسابقات بالظبط — مفيش تشغيل أوتوماتيك).

## الـ Flow الكامل

```
1. الأدمن: يضيف المسابقة + بنود التقييم الخاصة بيها (criteria)
2. الأدمن: يضيف المسابقة للجدول (وقت + مكان)
3. الأدمن: يضغط "تشغيل" يدوياً لما المسابقة تبدأ
   → التقارير اللي اترفعت على المسابقة دي تظهر للحكام
4. الحكام (judges): يقرأوا التقرير + يحطوا درجة لكل بند (0 → max)
   → Submit → الدرجات بتترفع للأدمن أوتوماتيك
5. الأدمن: يضغط "إنهاء" المسابقة
   → Leaderboard يتحدث بالمسابقة دي
6. الـ Leaderboard الكلي: مجموع درجات كل فريق في كل المسابقات
```

---

## Roles

| Role | Access |
|---|---|
| `team` | رفع التقارير (نفس flow الحالي) |
| `judge` | صفحة الحكام — يشوف تقارير المسابقات اللي شغالة بس + يقيّم |
| `admin` | كل حاجة (الجدول + بنود التقييم + التقارير + Leaderboard + toggles) |
| `executor` | ❌ مفيش — منفذي البرامج تواصلهم هاتفي مع الأدمن |

---

## الـ Leaderboard

- **الظهور**: عند الأدمن بس
- **الـ Home page (public)**: يعرض top 3 + باقي الأوائل من الـ leaderboard الكلي
- **التحديث**: بعد ما المسابقة تخلص (مش real-time — عشان مفيش ضغط على الحكام ومفيش confusion)

---

## Reports Flow (مرجع سريع)

- التقارير تظهر للحكام **بس وقت ما المسابقة شغالة**
- الـ AI بيفرز التقارير: يربط كل تقرير بالمسابقة الصح بناءً على الـ metadata (اسم المسابقة اللي الفريق اختارها من dropdown)
- لو الفريق اختار مسابقة غلط → flag للأدمن

## ملاحظات التنفيذ (للمطور)

- `competitions` table: فيها `criteria_json` (array of `{name, max_score}`)
- `submissions` table: `team_id`, `competition_id`, `file_url`, `submitted_at`
- `scores` table: `submission_id`, `judge_id`, `criterion_name`, `score`, `submitted_at`
- `judge_assignments` table (لو محتاجين): `competition_id`, `judge_id`
- `leaderboard` view: aggregate من scores + competitions
